Normalization in sql
  

database normalization is a technique of organizing in the database 
in order to avoid data redundancy , avoid undesirable characteristics and it will be difficult to handle database without normalization it will become inaccurate, slow , insufficient 
  types of Normalization
 1NF , 2NF , 3NF

1) 1NF -  the data  in each column should have an atomic ( single value ) 

 in simple words if one table is there and it is having 3 columns and in the 3rd column data is more than one and we have to apply first normal form to it then the result is going to be like it will create more rows with the same initial name and diffrent rows for diffrent data so it will become readable and easy 

 * without this it will become difficult to handle data 


2) 2NF - All the rules of first normal form and it will make the table data to a diffrent table like same data is there with same keyword then it will create diffrent data for it 


3) 3NF - All the rules of 2nd normal form and it should not be dependant on primary key

SQL like operator 

by using this operator we can easily display data from a  column of a table with specific character as starting or ending

for ex.

select * from employee

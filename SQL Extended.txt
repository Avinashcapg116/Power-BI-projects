2) Left Join

in this join all values of left table will display and matching values from right table


Select * 
from customer As c
Left Join Payment As p
On  c.ID = p.ID


result will be it will display all the records from left table i.e. table customer  c and matching records of payment p and remaining records of payment id will display null

for ex. id 1 2 3 will display with all the data of both the columns but id 4 will display for payment column


3) Right Join


Select * 
from customer As c
Right Join Payment As p
On  c.ID = p.ID


same as left join but it will display right column values 



4) Full Outer Join


returns all records when there is a match in either left or right table



Select * 
from customer AS c
Full Outer join payment as p
ON c.ID = p.ID


It will be display all the id and of both the columns with whatever data null or not still it will display in it

Window Function In SQL

Analytics function

select DeptName, max (salary) as maxsalary
from employee
group by DeptName; 


here we can display the max from each department using this query but in such case if we want to display all records of the same then we will use the windows function

select e.*,
max (salary) over () as maxsalary    ---- here over is the window function created 
from employee e;

select e.*,
max (salary) over (partition by deptname) as maxsalary --- partition by each department and alias maxsalary will display max salary of each dept
from employee e;

------------------------------row number function
select e.*,
row_number () over (Order By e.Salary) as rn   -- in row number function it is not possible as max we have to mention order by to execute the query
from employee e;


SELECT e.*,
       ROW_NUMBER() OVER (PARTITION BY deptname ORDER BY e.ID) AS rn
FROM employee e;

------------------------SET operators-------------------------------

1) Union 2) Union all 3) Intersect 4) Except 

the result sets of all queries must have the same number of columns
in  every result set the data type of each column must match the data type of its  corresponding column in final result set

1) Union - it combines two or more result sets in a single set without duplicates

use AdventureWorks2022



-- Union operator --
select CurrencyCode --109 rows
from Sales.CountryRegionCurrency
Union
select CurrencyCode -- 105 rows
from Sales.Currency

this combines two select statements but displays only distinct values 

--Union all operator ---
select CurrencyCode --109 rows

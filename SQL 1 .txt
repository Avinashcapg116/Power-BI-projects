Normalization in sql
  

database normalization is a technique of organizing in the database 
in order to avoid data redundancy , avoid undesirable characteristics and it will be difficult to handle database without normalization it will become inaccurate, slow , insufficient 
  types of Normalization
 1NF , 2NF , 3NF

1) 1NF -  the data  in each column should have an atomic ( single value ) 

 in simple words if one table is there and it is having 3 columns and in the 3rd column data is more than one and we have to apply first normal form to it then the result is going to be like it will create more rows with the same initial name and diffrent rows for diffrent data so it will become readable and easy 

 * without this it will become difficult to handle data 


2) 2NF - All the rules of first normal form and it will make the table data to a diffrent table like same data is there with same keyword then it will create diffrent data for it 


3) 3NF - All the rules of 2nd normal form and it should not be dependant on primary key

SQL like operator 

by using this operator we can easily display data from a  column of a table with specific character as starting or ending

for ex.
select * from employee               --------- here % is added after P in order to display cities starting with letter P
where City Like 'P%';

select * from employee
where city like '%i';                  --------- here % is added before i in order to display cities ending with letter I


select * from employee
where EmpName like '_a%';              --------- here _ and % both are used in order to display a as second character in employees names

select * from employee
where EmpName not like '_a%';         


same thing we can use for city , salary also

select * from employee
where Salary like '_0%';


select * from employee
where City like '_u%';

select * from employee
where City like '__m%';          ----  here its displaying cities which are having m as 3rd letter so we can add no. of __ to display desired value

*)------------------------------Stored procedure--------------------important topic

when we use a query again and again then we use stored procedure


for ex. 

use NEWDB3;


select * from employee;


create procedure AllRecords 
AS
Select * from employee
Go;                               ----- simple stored procedure where we have created procedure to select entire table 

exec AllRecords;



create procedure AllRecords2 @City varchar(20)
AS
Select * from employee             ------ here we have added some new elements like city 
where City= @City;

exec AllRecords2 @City = 'Pune'


create procedure AllRecords4 @City varchar(255), @EmpName varchar(255)
AS
Select * from employee 
where City = @City
AND
EmpName = @EmpName;                    ------ here we have more than one elements to showcase using sp
